Métricas de Entrenamiento

Para el entrenamiento del modelo, primero se normalizaron los datos de los estados, municipios y la fecha.

Para la fecha, separé el año y los meses. En el caso de los meses, utilicé funciones seno y coseno para generar características que permitieran al modelo entender la naturaleza periódica de estos datos. Para el año, apliqué una normalización estándar.

Debido a la gran cantidad de muestras en los municipios, decidí usar una capa de embedding para no incrementar la complejidad del modelo. Para ello, utilicé un LabelEncoder que transforma etiquetas categóricas en valores numéricos, y posteriormente pasé estos valores por la capa de embedding.

Observaciones

El modelo generaliza bien, como se puede observar en la curva de pérdida. Sin embargo, al analizar la gráfica de valores reales vs. predichos, se nota que hay algunos valores que el modelo no predice con buena exactitud, especialmente cuando los datos tienen valores más altos.

Considero que este proyecto tiene margen de mejora al probar con las siguientes metodologías:

  • Enfocarse en un solo estado. Existe mucha variabilidad en el índice de violencia según el estado, y factores como economía, población, clima, costumbres y festividades son únicos. Aislar el problema por estado puede ayudar a capturar mejor las tendencias.
  • Relacionando con el punto anterior, incluir otros datos de entrada que tengan correlación con el índice de violencia podría aumentar la precisión del modelo.
  • Acontecimientos como la pandemia de COVID-19 también afectan los resultados. El modelo actual no toma esto en cuenta, por lo que sería importante incorporar este tipo de eventos.
  • Probar diferentes arquitecturas de redes neuronales puede ayudar a comparar y validar mejoras en el rendimiento.
